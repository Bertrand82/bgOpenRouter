404 Causes possibles et solutions

Si vous avez restreint les fournisseurs autorisés dans vos paramètres OpenRouter, cela peut entraîner une erreur 404 si aucun fournisseur disponible ne correspond à votre demande. Pour résoudre ce problème :
  1. Accédez à vos paramètres sur OpenRouter.preferences dans la section "Allowed Providers", autoriser le provider voulu.
  2. Faites défiler jusqu'à la section des fournisseurs.
  3. Débloquez les fournisseurs ignorés et effacez les restrictions sur les fournisseurs autorisés.
  4. Enregistrez les modifications.

403 Causes possibles et solutions

L'erreur que vous rencontrez, « Http failure response for https://openrouter.ai/api/v1/chat/completions: 403 OK », indique que le serveur a compris votre requête mais refuse de l'autoriser. Voici les causes possibles et les solutions correspondantes :
  • Modèle nécessitant une clé API spécifique:Cause : Certains modèles, comme o3, requièrent l'utilisation de votre propre clé API OpenAI.
  • Clé API invalide ou expirée
  • Quota de crédits dépassé
  • Restrictions géographiques : L'accès à certains modèles ou services est restreint dans certaines régions.
  • Modération de contenu Cause : Votre message a été signalé par le système de modération d'OpenRouter.
